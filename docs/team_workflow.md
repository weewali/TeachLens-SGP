# Team Workflow

## Basic Git Workflow

1. Pull the latest version before working.
2. Work only inside your assigned folder or file.
3. Save changes.
4. Commit with a clear message.
5. Once your branch is ready, make a pull request

## Example Commit Messages

```text
Add instructor video source list
Add student video source list
Create annotation guidelines
Add frame extraction notes
Update dataset folder structure
Add first annotation review
```

## Storage Rule

Do not commit full videos, extracted frames, full datasets, trained model weights, or generated outputs. Use GitHub for code, documentation, configuration, and small samples only. For any questions or confusions please ask Ali Zainal.
